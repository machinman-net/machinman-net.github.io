ctoohtml Development History

Version 1.5, Nov 17, 2010

  • Suppress multiple anchor line.

Version 1.4, Nov 22, 2009

  • No more C++ support,
  • add code colourization,
  • add cross reference page and links
  • merge ctoohtml and ctoohtml.pl into one file,
  • add html insertion for header, top and bottom of pages,
  • command line options parsing with getopt,
  • change options letters
Emmanuel Azencot

Version 1.3, June 30, 1996

  • Fixed some bugs in the TAGS parsing routine.

Version 1.2.1, April, 1996

  • This is an adaptation of v1.2 by Jerome Chouinard.

Version 1.2, December 20, 1995

  • Official release of ctoohtml
  • Added -C option to indicate that .c and .h files are to be interpreted as C++ files and not c files.

Version 1.1, August 7, 1995

  • Sources files may now include the special directive: 
    [DEFINE DESCRIPTION = <strg>]
    where <strg> is a string that describes the contents of the file. This string is used in generating the source index file.

Version 1.0, July 31, 1995

  • Fixed bubble sort so that "tag class" information is swapped correctly.
  • Fixed bug that occurs when more than one overload is detected for any one tag.

Version 0.2

  • Added the ability to write HTML markup directly into the C or C++ files.
  • Added the ability to handle overloaded functions. Unfortunately, the best we can do using this mechanism is to flag the found token and present hyperlinks for all the possible matches.
  • Fixed the bug that kept #include hyperlinks from being generated when the file name is specified with <>.
  • Added #define macro capability
  • Now generates several indices: files.html (list of all files processed), tags.all.html (sorted list of all tags), and tags.file.html (sorted by source file).
  • Added enum tag types.
  • Made string recognition more robust.
  • Added Support for dealing with NSL code. -n option
  • Added ability to specify files outside of the local directory.
  • Added the ability to specify a destination directory other than the local one. -z option

Version 0.1

  • Anchoring of tags.
  • Automatic hyperlinking of tag references.
Andrew H. Fagg
